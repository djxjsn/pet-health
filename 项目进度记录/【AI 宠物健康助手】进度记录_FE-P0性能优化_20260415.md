## 二、各任务详细记录

### P0-1: 集成 next/image 组件

**目标**: 将项目中所有 `<img>` 标签替换为 `next/image`，实现自动格式转换、响应式尺寸和懒加载。

**交付产物**:
- `src/components/OptimizedImage.tsx` — 统一图片优化组件（支持 data URL 回退）
- 6 张商品占位图片 (`public/images/products/*.jpg`)
- 3 处替换：MessageBubble / Shop / ImageUploader

**技术实现**:
```
OptimizedImage 架构:
┌──────────────────────────────────────┐
│          OptimizedImage              │
├──────────────────────────────────────┤
│ 输入: src, alt, width, height, fill │
│       priority, sizes, quality      │
│       placeholder, fallback          │
├──────────────────────────────────────┤
│ 判断:                                │
│ • isDataUrl / isBlobUrl → 原生 img   │
│ • imgError + fallback → 显示降级     │
│ • 正常情况 → <Image> (next/image)    │
│   ├─ fill 模式: fill + sizes         │
│   └─ 固定模式: width + height        │
│       自动: AVIF > WebP > 原格式     │
└──────────────────────────────────────┘
```

**关键特性**:
- **data URL 兼容**: ImageUploader 的 FileReader 预览自动回退到原生 `<img>`
- **错误降级**: 加载失败显示 fallback 内容（如 emoji 占位符）
- **fill 模式**: Shop 页面商品图使用 `fill` + `sizes` 实现响应式
- **priority 属性**: 首屏前 4 个商品设置 `priority={true}` 提升优先级

## 四、解决的关键问题

本阶段共解决 **6 个关键技术问题**：

1. **memo() 导出模式兼容性** — Next.js 构建不识别 `export default memo(function(){})`，改为分离式导出
2. **类型系统统一** — MessageBubble 本地接口与全局 types 冲突，使用 `ExtendedMessage extends Message` 扩展
3. **PageProps 限制** — Next.js 14 页面组件不接受自定义 className prop
4. **placeholder 类型安全** — 条件分支替代 spread 解决 `string` vs `PlaceholderValue` 冲突
5. **headers undefined 安全** — API utils 中 options.headers 可能为空
6. **Dynamic 导出匹配** — `.then(mod => mod.default)` vs `.then(mod => mod.Xxx)` 区分
