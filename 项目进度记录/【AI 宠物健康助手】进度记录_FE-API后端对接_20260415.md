### API-2: WebSocket 连接管理

**目标**: 重写 WebSocket Hook，完全匹配后端的 WS 协议规范。

**交付产物**: `src/hooks/useWebSocket.ts` (~280 行)

**协议实现细节**:

| 协议步骤 | 客户端行为 | 服务端预期 | 状态 |
|----------|-----------|-----------|------|
| 1. 建立连接 | new WebSocket(url) | - | connecting |
| 2. 认证 | `{ type: "auth", token }` | 返回 connected | → connected |
| 3. 心跳 | 每25s 发 `{ type: "heartbeat" }` | 回复 heartbeat | 保持活跃 |
| 4. Ping/Pong | 收到 ping → 回复 pong | 定期发送 ping | 保活 |
| 5. 发消息 | `{ type: "message", data: {...} }` | 处理中+回复 | 正常对话 |
| 6. 断连处理 | 自动重连(指数退避,最多5次) | - | reconnecting→error |

**关键特性**:
- **自动认证**: 连接建立后自动发送 token 进行 auth 握手
- **心跳保活**: 25s 间隔双向心跳，防止中间件断开
- **智能重连**: 指数退避策略 (2s → 3s → 4.5s → ...)，上限 5 次
- **状态管理**: idle/connecting/connected/disconnected/error 五态机
- **消息收发**: 用户消息自动加入 messages 数组，助手消息通过 onMessage 回调

### API-3: 全局错误处理体系

**目标**: 建立分层错误分类、统一处理、用户友好提示的完整错误体系。

**交付产物**: `src/utils/error.tsx` (~260 行)

**错误分类体系 (8 类)**:

| 分类 | HTTP 范围 | Toast 类型 | 自动动作 |
|------|-----------|------------|---------|
| NETWORK | 无响应 | warning (6s) | 显示"检查网络" |
| AUTH | 401/403 | error (8s) | 401→跳转登录页 |
| SERVER | 5xx | error (8s) | 提供"重试"按钮 |
| VALIDATION | 4xx | warning (4s) | 显示"检查输入" |
| NOT_FOUND | 404 | info (3s) | 显示"内容不存在" |
| TIMEOUT | AbortError | warning (5s) | 显示"稍后重试" |
| PARSE_ERROR | SyntaxError | error (6s) | 显示"联系技术支持" |
| UNKNOWN | 其他 | error (5s) | 显示"刷新页面" |

**三大核心能力**:

1. **handleAppError(error, context?)**: 统一入口函数
   - 错误分类 → 日志输出 → Toast 通知 → Auth 特殊处理
   - 可选 context 参数标注错误来源上下文

2. **withErrorHandling(fn, options?)**: 异步操作包装器
   - 自动 try/catch + 错误分类 + Toast
   - 支持 fallback 默认值返回
   - 支持 onError 自定义回调

3. **ErrorBoundary React 组件**:
   - 捕获子组件渲染异常
   - 展示错误 UI + 重试按钮 + 返回首页
   - 不影响其他组件正常渲染
