## 二、问题发现与解决

### 问题1: 数据库Schema不同步 (Critical)
- **现象**: 注册用户时 `OperationalError: Unknown column 'user_profiles_1.pet_count'`
- **根因**: `UserProfile` 模型定义了 `pet_count` 字段，但 Alembic 迁移脚本未包含
- **解决**: 执行 SQL `ALTER TABLE user_profiles ADD COLUMN pet_count INT NOT NULL DEFAULT 0`
- **解决文件**: `animal/scripts/fix_schema.py`
- **状态**: ✅ 已解决

### 问题2: LLM服务不可用 (Environment)
- **现象**: Chat/Health/Behavior 端点请求超时
- **根因**: 后端环境无法访问 HuggingFace，SentenceTransformer 模型加载失败
- **影响**: 仅影响LLM推理端点，CRUD/认证端点完全正常
- **状态**: ⚠️ 环境限制，非代码问题

## 六、面试知识沉淀

### 核心技术亮点

**1. API层设计（可面试点）**
- 统一ApiClient封装：自动重试（仅5xx）、超时AbortController、Token注入
- 错误分类体系：8种ErrorCategory，精准映射HTTP状态码到用户消息
- OAuth2密码流实现：FormData格式登录，JWT token管理

**2. WebSocket状态机设计（可面试点）**
- 完整状态机：idle→connecting→connected→disconnected→error
- 指数退避重连：base 2s × 1.5^n，最大5次
- 协议对齐：auth握手+heartbeat+ping/pong+流式响应

**3. 错误处理三层防护（可面试点）**
- 请求层：withErrorHandling包装器
- 应用层：handleAppError（Toast+401跳转）
- 渲染层：ErrorBoundary React组件

**4. 数据库问题排查（可面试点）**
- Alembic迁移脚本与SQLAlchemy模型不一致的排查方法
- 生产数据库Schema变更的 ALTER TABLE 修复方案
