## 技术亮点

### 1. JWT 双令牌机制
**实现方式**:
- Access Token: 短期令牌（30分钟），用于API访问
- Refresh Token: 长期令牌（7天），用于刷新access token
- 令牌类型标识: payload中包含 `type` 字段区分

**优势**:
- 减少用户重复登录
- 提高安全性（access token 泄露影响有限）
- 支持令牌撤销机制（未来扩展）

### 2. OAuth2 标准密码流
**实现方式**:
- 使用 `OAuth2PasswordRequestForm` 接收表单数据
- 返回标准 OAuth2 响应格式
- 支持 Swagger UI "Authorize" 按钮

**优势**:
- 符合行业标准
- 兼容第三方OAuth2客户端
- 自动生成API文档

### 3. bcrypt 密码哈希
**实现方式**:
- 使用 bcrypt 算法
- rounds=12（2^12次迭代）
- 自动加盐

**优势**:
- 抗彩虹表攻击
- 抗暴力破解
- 安全性业界认可

### 4. 双因子登录
**实现方式**:
- 支持手机号登录
- 支持邮箱登录
- 自动识别登录类型

**优势**:
- 提升用户体验
- 增加账号安全性
- 灵活性强

## 遇到的问题与解决方案

### 问题 1: Pydantic 配置验证错误
**现象**: `ValidationError: Extra inputs are not permitted`

**原因**: Pydantic v2 默认不允许额外字段，`.env` 文件中包含占位符 API Key

**解决方案**:
```python
model_config = {
    "extra": "ignore",  # 忽略额外字段
}
```

**结果**: ✅ 问题解决

---

### 问题 2: 数据库连接 Unicode 编码错误
**现象**: `UnicodeEncodeError: 'latin-1' codec can't encode characters`

**原因**: MySQL 连接使用 latin1 编码，无法处理中文密码

**解决方案**:
```bash
# 修改 .env 文件使用英文密码
DATABASE_URL=mysql+pymysql://root:test_password@localhost:3306/ai_pet_health
```

**结果**: ✅ 问题解决

---

### 问题 3: 测试环境数据库隔离
**现象**: 测试尝试连接真实 MySQL 数据库

**原因**: `conftest.py` 导入 `main.py` 时触发数据库连接

**解决方案**:
```python
# 在导入 app 之前设置测试环境变量
os.environ["DATABASE_URL"] = "sqlite:///:memory:"
os.environ["TESTING"] = "True"
```

**结果**: ✅ 测试完全隔离

---

### 问题 4: 缺失导入函数
**现象**: `NameError: name 'get_user_by_id' is not defined`

**原因**: `auth.py` 中缺少 CRUD 函数导入
