## 四、问题与解决（v5.0新增）

### 4.1 本阶段发现的问题

| # | 问题描述 | 类型 | 影响 | 解决状态 |
|---|---------|------|------|---------|
| 1 | 数据库Schema缺少pet_count列 | Critical | 注册失败 | ✅ 已修复 |
| 2 | HuggingFace网络不可达 | Environment | LLM端点超时 | ⚠️ 降级处理 |
| 3 | OAuth2 FormData格式问题 | Bug | 登录400错误 | ✅ 已修复 |
| 4 | Uvicorn热重载崩溃 | Environment | CI测试中断 | ✅ 已绕过 |

### 4.2 问题1详细：数据库Schema不同步

**现象**：`OperationalError: Unknown column 'user_profiles_1.pet_count'`

**根因**：SQLAlchemy `UserProfile` 模型定义了 `pet_count` 字段，但 Alembic 迁移脚本未同步

**解决**：执行 `ALTER TABLE user_profiles ADD COLUMN pet_count INT NOT NULL DEFAULT 0`

**文件**：`animal/scripts/fix_schema.py`

**经验沉淀**：模型变更后必须同步更新 Alembic 迁移脚本

### 4.3 问题2详细：LLM服务不可用

**现象**：Chat/Health/Behavior 端点超时（HuggingFace连接超时）

**根因**：后端环境无法访问 HuggingFace（网络限制）

**影响**：仅LLM推理端点不可用，CRUD/认证端点完全正常

**降级策略**：
- API层超时正确分类为 `TIMEOUT` ErrorCategory
- 前端Toast正确展示超时提示
- 非LLM端点完全正常
