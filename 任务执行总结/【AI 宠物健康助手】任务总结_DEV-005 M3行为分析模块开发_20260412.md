## （一）项目解决问题的过程

### 1. 具体问题

需要实现 M3 行为分析模块，提供宠物行为分析、原因推断、训练建议、行为历史查询等功能。核心挑战是如何结合品种特性、年龄因素和行为知识库生成准确的分析结果和可操作的训练建议。

### 2. 分析过程

- **需求分析**：
  - 行为分析需要识别行为类别（拆家、嚎叫、攻击等）
  - 原因分析需要考虑品种特性、年龄阶段、环境因素
  - 训练建议需要具体可操作，包含方法和时长
  - 需要支持行为历史记录和追踪

- **架构设计**：
  - 沿用已有的分层架构（Model → Schema → CRUD → API）
  - BehaviorAnalysisTool 作为核心编排工具
  - TrainingRecommendationTool 独立实现训练建议生成
  - 内置品种特性数据库（BREED_CHARACTERISTICS）
  - 行为类别自动检测（BEHAVIOR_KEYWORD_MAP）

- **技术选型**：
  - 内置品种知识库：覆盖 10+ 常见犬种、8+ 常见猫种
  - 关键词自动检测：BEHAVIOR_KEYWORD_MAP 实现行为类别归类
  - LangChain ChatPromptTemplate + StrOutputParser 实现 LLM 调用
  - ChromaDB 向量数据库实现知识检索
  - 年龄阶段分析：幼年期/青少年期/成年期/老年期

### 3. 解决方案及结果

- **方案**：
  1. 数据库层：创建 BehaviorAnalysis 模型，支持行为分析记录存储
  2. 品种知识库：内置 BREED_CHARACTERISTICS 字典，包含能量级别、性格特征、常见问题、运动需求
  3. 行为检测：BEHAVIOR_KEYWORD_MAP 实现关键词→行为类别映射
  4. 工具层：
     - BehaviorAnalysisTool：编排完整分析流程（档案→品种分析→年龄分析→知识库检索→LLM分析→严重度评估）
     - TrainingRecommendationTool：生成品种特定训练建议和方法
  5. API 层：4 个 RESTful 端点，统一权限校验和异常处理

- **结果**：
  - 4 个 API 端点全部实现
  - 2 个核心工具全部实现
  - 内置品种知识库（18+ 品种）
  - 5 种行为类别支持（destructive/howling/aggression/food_refusal/excessive_licking）
  - 测试用例 40+ 个，覆盖率 >90%

### 4. 经验教训

- **品种知识库内置**：避免外部依赖，内置常见品种的行为特征数据，响应更快
- **行为类别自动归类**：BEHAVIOR_KEYWORD_MAP 实现关键词到类别的自动映射，减少用户输入负担
- **年龄因素考量**：不同年龄段宠物行为原因和处理方式不同，需要针对性分析
- **训练建议具体化**：TRAINING_PLANS 包含具体方法、描述和见效时长，提高可操作性

## （四）原理逻辑

### 1. 知识点：品种知识库设计

**原理**：将品种特性数据以字典结构存储，每个品种包含：
- `energy`：能量级别（极高/高/中/低）
- `traits`：性格特征列表
- `common_issues`：常见行为问题
- `exercise_need`：建议运动量

**项目应用**：
```python
BREED_CHARACTERISTICS = {
    "dog": {
        "哈士奇": {
            "energy": "极高",
            "traits": ["精力旺盛", "独立", "爱嚎叫", "固执"],
            "common_issues": ["拆家", "嚎叫", "逃跑", "破坏家具"],
            "exercise_need": "每日至少2小时高强度运动",
        },
        ...
    }
}

def get_breed_info(species: str, breed: Optional[str]) -> Optional[Dict]:
    # 精确匹配 + 模糊匹配
    species_data = BREED_CHARACTERISTICS.get(species, {})
    if breed in species_data:
        return species_data[breed]
    for breed_name, info in species_data.items():
        if breed_name in breed or breed in breed_name:
            return info
    return None
```

### 2. 知识点：行为类别自动检测

**原理**：通过关键词匹配将用户自由文本描述归类到标准行为类别。

**项目应用**：
```python
BEHAVIOR_KEYWORD_MAP = {
    "destructive": ["拆家", "咬坏", "破坏", "啃咬", ...],
    "howling": ["嚎叫", "吠叫", "叫唤", "哀嚎", ...],
    "aggression": ["攻击", "咬人", "凶", "护食", ...],
    "food_refusal": ["拒食", "不吃", "不吃饭", ...],
    "excessive_licking": ["过度舔毛", "舔爪", ...],
}

def detect_behavior_category(description: str) -> str:
    for category, keywords in BEHAVIOR_KEYWORD_MAP.items():
        for kw in keywords:
            if kw in description:
                return category
    return "other"
```

### 3. 知识点：年龄阶段行为分析

**原理**：不同年龄段的宠物有不同的行为特征和问题，需要针对性分析。

**项目应用**：
```python
def calculate_pet_age_months(birth_date: Optional[date]) -> Optional[int]:
    if not birth_date:
        return None
    today = date.today()
    months = (today.year - birth_date.year) * 12 + (today.month - birth_date.month)
    return max(0, months)

def _analyze_age_factor(age_months, behavior_description):
    if age_months < 6:
        stage = "幼年期"
        factors = ["幼宠处于社会化关键期", "换牙期啃咬正常"]
    elif age_months < 18:
        stage = "青少年期"
        factors = ["精力旺盛期", "拆家概率最高"]
    elif age_months < 84:
        stage = "成年期"
        factors = ["行为相对稳定"]
    else:
        stage = "老年期"
        factors = ["可能出现认知功能下降", "老年认知障碍(CDS)"]
    return {"stage": stage, "age_months": age_months, "factors": factors}
```
