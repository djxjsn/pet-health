## 二、详细风险评估

### 2.1 R5 - LLM API调用成本风险

**风险描述**:  
项目依赖外部LLM API进行核心功能（健康咨询、行为分析、知识检索），随着用户量增长，API调用费用可能超出预算。

**影响分析**:
- **直接影响**: 运营成本增加
- **间接影响**: 可能需要限制功能或降低响应质量
- **财务影响**: 月度API费用预估 ¥500-5000（取决于用量）

**发生概率**: 40%（中等）

**当前缓解措施**:
- [x] 实现结果缓存机制（ToolExecutor缓存）
- [x] 优先使用本地知识库(RAG)减少LLM调用
- [x] 分级响应：简单问题用规则引擎，复杂问题用LLM
- [ ] 待实施：设置月度费用预警阈值
- [ ] 待实施：探索开源模型替代方案

**监控指标**:
- 每日API调用次数
- 单次调用平均成本
- 月度累计费用

**建议行动**:
1. 短期(1周内): 设置费用监控Dashboard
2. 中期(1个月内): 评估开源模型(Qwen/ChatGLM)可行性
3. 长期: 根据实际使用量调整定价策略

---

### 2.2 R6 - 前端LLM推理端点不可用

**风险描述**:  
前端对话页面依赖后端LLM推理端点，若API Key失效或服务不可用，核心对话功能将无法使用。

**影响分析**:
- **用户体验**: 对话功能完全不可用
- **业务影响**: 核心价值主张受损
- **技术影响**: 需要优雅降级方案

**发生概率**: 30%（中高）

**当前缓解措施**:
- [x] 后端实现fallback机制（无LLM时返回知识库答案）
- [ ] 前端需添加错误提示和重试逻辑
- [ ] 配置多API Key轮换机制

**监控指标**:
- API可用性(uptime)
- 平均响应时间
- 错误率(error rate)

**建议行动**:
1. 立即: 配置有效的API Key并测试
2. 本周内: 前端添加"AI助手暂时不可用"的友好提示
3. 本月内: 实现API Key自动轮换

### 2.4 R9 - 数据安全合规性待验证 [新增]

**风险描述**:  
DEV-009数据安全模块尚未开发，当前系统在数据加密、访问控制、审计日志等方面存在合规性缺口。

**影响分析**:

- **安全风险**: 数据泄露或未授权访问
- **信任风险**: 用户对产品安全性信心不足

**发生概率**: 20%（低，因尚未正式运营）

**当前状态**:
- [ ] 数据传输加密(TLS) - 部分实现(FastAPI默认)

- [ ] 访问权限控制 - 基础JWT，无RBAC
- [ ] 审计日志 - 未实现
- [ ] 数据脱敏 - 未实现

**解决计划**: 
- **DEV-009模块**将全面解决此风险
- 预计工作量: ~1周
- 建议优先级: P2（可在v1.0之后迭代完善）
