## 一、项目概览

### 1.1 项目目标

基于 LangChain 框架开发 AI 宠物健康助手，提供宠物健康咨询、疾病诊断、营养建议等功能，帮助宠物主人更好地照顾宠物健康。

### 1.2 技术栈

- **后端**：Python 3.12.10 + FastAPI 0.135.3 + SQLAlchemy 2.0
- **数据库**：MySQL 8.0+（开发环境使用 SQLite 内存数据库测试）
- **AI 框架**：LangChain + ChromaDB（向量数据库）
- **认证**：JWT（HS256）+ bcrypt 密码哈希
- **测试**：pytest + pytest-cov
- **前端**：Next.js 14 + TypeScript（待开发）

### 1.3 项目周期

- **项目总周期**：16周（2026年4月15日 - 2026年8月4日）
- **当前阶段**：第1周（2026-04-08 ~ 2026-04-12）

## 五、风险与问题

### 5.1 已解决的问题

1. **依赖包缺失**
   - 问题：测试环境缺少 pytest、fastapi 等依赖
   - 解决：通过 pip install 安装所有必要依赖
   - 状态：✅ 已解决

2. **用户数据隔离**
   - 问题：需要确保用户只能操作自己的宠物数据
   - 解决：在 API 层校验 pet.user_id 与 current_user.user_id 一致性
   - 状态：✅ 已解决

3. **向量数据库模型下载失败**
   - 问题：SentenceTransformer 模型因网络问题无法下载
   - 解决：实现降级策略，切换到 DefaultEmbeddingFunction
   - 状态：✅ 已解决

4. **WebSocket 连接稳定性**
   - 问题：长连接易受网络波动影响
   - 解决：实现 30 秒心跳检测机制，ConnectionManager 管理连接
   - 状态：✅ 已解决

5. **LLM API 调用失败**
   - 问题：外部 LLM API 可能不稳定
   - 解决：实现降级方案，基于知识库和规则返回基本建议
   - 状态：✅ 已解决

### 5.2 潜在风险

1. **LLM API 调用成本高**
   - 影响：预算超支
   - 概率：中
   - 应对措施：使用 DeepSeek 等低成本替代，实现缓存机制

2. **RAG 检索准确率不达标**
   - 影响：用户体验差
   - 概率：中
   - 应对措施：多轮测试优化，混合检索策略

3. **开发进度延期**
   - 影响：交付延迟
   - 概率：低
   - 应对措施：每日站会跟踪，及时调整优先级
