# Superpowers

Superpowers is a framework that transforms AI coding agents into disciplined software engineers by enforcing non-negotiable workflows and best practices.

## When to Use This Skill

Use this skill when the user:
- Needs to build complex software projects
- Wants to follow proper software engineering practices
- Needs help with project planning and task decomposition
- Wants to implement test-driven development
- Needs structured code review processes
- Is working on enterprise-grade software

## Core Principles

1. **Design Before Code**: Require architecture and design documents before implementation
2. **Tests Before Features**: Enforce test-driven development practices
3. **Structured Review**: Implement code review between every task
4. **Task Decomposition**: Break complex projects into manageable tasks
5. **Quality Assurance**: Ensure code quality through linting and testing

## Workflow

### Phase 1: Project Planning
- Requirements analysis
- Architecture design
- Task decomposition
- Timeline estimation

### Phase 2: Development
- Test case creation
- Implementation
- Code review
- Quality assurance

### Phase 3: Delivery
- Integration testing
- Deployment preparation
- Documentation
- Knowledge transfer

## Key Features

- **Task Management**: Break down complex projects into manageable tasks
- **Test Automation**: Generate and run tests automatically
- **Code Review**: Provide structured code review processes
- **Quality Control**: Enforce code quality standards
- **Documentation**: Generate project documentation

## Usage Example

```
superpowers init project-name
superpowers plan "Build a web application with React and Node.js"
superpowers task add "Create authentication system"
superpowers task add "Implement user dashboard"
superpowers task start 1
superpowers test
superpowers review
superpowers task complete 1
superpowers task start 2
```

## Installation

To install this skill:
```bash
git clone https://github.com/obra/superpowers
cd superpowers
npm install
```

## Best Practices

- Always start with a clear project plan
- Break down large tasks into smaller subtasks
- Write tests before implementing features
- Conduct code reviews for every significant change
- Document design decisions and architecture
- Use version control effectively
- Follow coding standards and best practices
