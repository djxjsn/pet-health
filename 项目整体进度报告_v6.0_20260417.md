## 三、DEV-006 知识管理模块详情（v6.0新增）

### 3.1 模块架构

```
知识管理模块 (DEV-006)
├── 数据层
│   ├── KnowledgeDocument (MongoDB模型)
│   └── KnowledgeRepository (数据访问层)
├── 业务层
│   ├── KnowledgeService (业务逻辑)
│   └── KnowledgeUpdateService (更新服务)
├── 核心引擎
│   ├── DocumentLoader (文档加载器)
│   ├── TextChunker (文本分块器)
│   ├── KnowledgeIndexer (索引构建器)
│   └── KnowledgeRetriever (知识检索器)
└── API层
    ├── knowledge.py (14个端点)
    └── schemas/knowledge.py (8个Schema)
```

### 3.2 核心功能

| 功能 | 说明 | 状态 |
|------|------|------|
| 文档管理 | CRUD操作、发布/归档 | ✅ |
| 文档加载 | TXT/MD/JSON多格式支持 | ✅ |
| 智能分块 | 固定/语义/重叠3种策略 | ✅ |
| 向量索引 | ChromaDB向量化存储 | ✅ |
| 语义检索 | 支持分类/标签过滤 | ✅ |
| 增量更新 | 文档变更自动检测 | ✅ |
| 批量操作 | 批量发布/索引 | ✅ |

### 3.3 API端点列表

| 端点 | 方法 | 说明 |
|------|------|------|
| /knowledge/search | POST | 知识语义检索 |
| /knowledge/documents | GET/POST | 文档列表/创建 |
| /knowledge/documents/{id} | GET/PUT/DELETE | 文档详情/更新/删除 |
| /knowledge/documents/{id}/publish | POST | 发布文档 |
| /knowledge/index/rebuild | POST | 重建索引 |
| /knowledge/index/update | POST | 增量更新 |
| /knowledge/index/stats | GET | 索引统计 |
| /knowledge/update/* | POST/GET | 更新管理（4个） |
