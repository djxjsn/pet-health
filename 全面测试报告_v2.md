### 3.2 失败用例详细分析（6 个失败 + 3 个 Error）

#### 失败 1：行为分析字段名不匹配
**测试**: `test_behavior_analyze`
**原因**: 测试期望字段 `possible_causes` 在根级别，但实际在 `analysis_result` 嵌套结构中
**影响**: 🟡 中 - 测试断言需更新
**修复建议**: 更新测试断言路径为 `response['analysis_result']['possible_causes']`

#### 失败 2：Pydantic 配置验证行为变化
**测试**: `test_config_model_validation`
**原因**: Pydantic V2 不再对枚举值严格抛出 ValidationError
**影响**: 🟢 低 - 仅测试逻辑问题

#### 失败 3：Celery 异步服务不可用
**测试**: `test_service_available_after_init_mock`
**原因**: Celery 需要 Redis Broker 才能正常工作
**影响**: 🟡 中 - 异步任务需要 Redis

#### 失败 4：健康咨询 Schema 字段缺失
**测试**: `test_symptom_analysis`
**原因**: 返回的 `SymptomAnalysisResponse` 缺少 `diagnosis_result` 字段
**影响**: 🟡 中 - 健康咨询 API 响应格式变更

#### 失败 5：多轮对话 AI 响应措辞差异
**测试**: `test_conversation_context_persistence`
**原因**: AI 返回 "多大了" vs 测试期望 "多岁了"
**影响**: 🟢 低 - 语义相同，措辞差异

#### 失败 6：工具注册 Schema 验证失败
**测试**: `test_get_tools_schema`
**原因**: `PetInfoTool` 需要 `db` 参数但未提供
**影响**: 🟡 中 - 工具注册需传入 mock db

#### Error 1-3：ChatOpenAI 属性缺失
**测试**: `test_state_initialization`, `test_state_clearing`, `test_tool_registration`
**原因**: `pet_health_agent.py` 模块未导出 `ChatOpenAI`
**影响**: 🟡 中 - 需要 LLM API Key 配置
