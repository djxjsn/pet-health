## 一、迁移背景与目标

### 1.1 背景
当前项目所有数据表（8张）均使用 MySQL 存储，但随着业务发展，部分数据模型（对话、健康记录、行为分析等）具有高度非结构化特性，使用文档型数据库 MongoDB 更为合适。

### 1.2 目标
- 将 5 张文档型数据表从 MySQL 迁移到 MongoDB
- 保留 3 张关系型数据表在 MySQL（users, user_profiles, pets）
- 实现 MySQL + MongoDB 混合架构
- 保证数据完整性和业务连续性

---

## 二、迁移范围

### 2.1 保留在 MySQL 的表（3张）

| 表名 | 模型 | 理由 |
|------|------|------|
| `users` | User | 用户认证、登录，强事务需求 |
| `user_profiles` | UserProfile | 用户档案，与 users 一对一关联 |
| `pets` | Pet | 宠物档案，与 users 外键关联 |

### 2.2 迁移到 MongoDB 的表（5张 → 5个集合）

| 原表名 | 目标集合 | 迁移策略 |
|--------|---------|---------|
| `conversations` | `conversations` | 保留独立集合 |
| `messages` | `messages` | 保留独立集合（不嵌套） |
| `health_records` | `health_records` | 直接迁移 |
| `consultations` | `consultations` | 直接迁移 |
| `behavior_analyses` | `behavior_analyses` | 直接迁移 |
