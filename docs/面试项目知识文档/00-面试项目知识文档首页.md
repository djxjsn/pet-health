## 使用说明

### 文档目的

本文档核心目的是通过"复盘→沉淀→优化"提升能力、储备面试素材。

### 填写规范

每次总结需包含 4 个核心模块：
1. **项目解决问题的过程** - 记录"遇问题→分析→解决"完整链路
2. **项目技术难点与亮点** - 突出个人技术贡献
3. **涉及的知识点** - 按技术领域分类梳理
4. **原理逻辑** - 挖掘核心知识点底层原理

### 更新频率

- 总结需在工作完成后 **24 小时内** 录入
- 不得拖延至项目结束集中补写
- 每周回顾维护，形成完整知识体系

### 面试讲解要点

- 每条总结需能流畅讲解
- 突出个人解决问题的能力
- 展示技术深度和广度
- 体现项目真实经验

## 面试常见问题索引

根据文档内容，预整理以下面试高频问题：

### 项目经验类
- 项目中遇到的最大技术挑战是什么？如何解决的？
- 在项目中你最有成就感的技术贡献是什么？
- 项目中的技术难点有哪些？如何突破的？

### 技术原理类
- JWT 认证的原理是什么？
- FastAPI 的依赖注入是如何实现的？
- bcrypt 密码哈希的原理是什么？
- OAuth2 认证流程是怎样的？

### 工程实践类
- 如何保证代码质量和测试覆盖率？
- 如何处理数据库连接和事务？
- 如何设计可扩展的 API 接口？
