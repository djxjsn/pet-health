**步骤2：实现 Pydantic Schema**
```python
class PetCreate(BaseModel):
    name: str = Field(..., min_length=1, max_length=100)
    species: str
    breed: Optional[str] = None
    
    @field_validator("species")
    @classmethod
    def validate_species(cls, v):
        allowed = ["dog", "cat", "bird", "rabbit", "hamster", "fish", "other"]
        if v.lower() not in allowed:
            raise ValueError(f"物种必须是: {allowed}")
        return v.lower()
```
- **关键点**：field_validator 实现业务验证，支持大小写不敏感

**步骤3：实现 CRUD 操作**
```python
def get_pets_by_user(db: Session, user_id: str, skip: int, limit: int) -> Tuple[List[Pet], int]:
    query = db.query(Pet).filter(Pet.user_id == user_id)
    total = query.count()
    pets = query.offset(skip).limit(limit).all()
    return pets, total
```
- **关键点**：返回元组（列表，总数），使用 offset/limit 分页

**步骤4：实现 API 端点**

```python
@router.post("/pets", response_model=PetResponse, status_code=201)

    pet_in: PetCreate,
    db: Session = Depends(get_db),
    current_user: User = Depends(get_current_user)

    return create_pet(db, pet_in, current_user.user_id)
```
- **关键点**：使用 Depends 注入 db 和 user，状态码 201

## （二）项目技术难点与亮点

### 1. 技术难点

**难点1：用户数据隔离**
- **难度等级**：⭐⭐⭐ (3/5)
- **原因**：宠物档案是多租户数据，必须确保用户 A 无法访问/修改/删除用户 B 的宠物数据
- **突破方法**：
  ```python
  @router.get("/pets/{pet_id}")
  def get_pet(pet_id: str, db: Session, current_user: User):
      pet = get_pet_by_id(db, pet_id)
      if not pet:
          raise HTTPException(404, "宠物不存在")
      if pet.user_id != current_user.user_id:
          raise HTTPException(403, "无权访问此宠物")
      return pet
  ```
- **创新点**：权限校验放在 API 层，CRUD 层保持纯粹的数据操作

**难点2：分页查询设计**
- **难度等级**：⭐⭐ (2/5)
- **原因**：需要同时返回数据列表和总数，前端需要总数进行分页显示
- **突破方法**：
  - CRUD 层返回 `Tuple[List[Pet], int]`
  - API 层组装为 `PetListResponse`，包含 items、total、page、page_size
  - 使用 SQLAlchemy 的 offset/limit 实现分页
- **创新点**：分页逻辑在数据库层完成，避免查询所有数据后再切片

**难点3：Enum 类型跨数据库兼容性**
- **难度等级**：⭐⭐ (2/5)
- **原因**：SQLAlchemy 的 Enum 在 MySQL 中创建真正的 ENUM 类型，在 SQLite 中则退化为 VARCHAR
- **突破方法**：
  - 在 Model 层使用 SQLAlchemy Enum
  - 在 Schema 层使用 Pydantic 进行严格验证
  - 不依赖数据库级约束，在应用层做验证
- **创新点**：跨数据库兼容时，在应用层做验证而非依赖数据库特性

### 2. 技术亮点

## （四）原理逻辑

### 1. 知识点：FastAPI 依赖注入系统

**原理**：FastAPI 的依赖注入基于 Python 的类型注解和 `Depends()` 函数。当请求到达时，FastAPI 按依赖链从最底层开始解析，将结果注入到端点函数参数中。依赖可以嵌套（如 `get_current_active_superuser` 依赖 `get_current_user`），形成依赖树。

**底层实现**：
1. FastAPI 在启动时扫描所有路由函数的参数
2. 识别 `Depends()` 标记的参数
3. 构建依赖图（有向无环图）
4. 请求到达时，按拓扑顺序执行依赖函数
5. 将结果缓存并注入到路由函数

**应用体现**：
```python
# 依赖链：get_db -> get_current_user -> get_current_active_user
@router.post("/pets")
def create_pet(
    db: Session = Depends(get_db),           # 子依赖: 数据库会话
    current_user: User = Depends(get_current_user)  # 子依赖: 当前认证用户
):
    # db和current_user都由框架负责创建和注入
    pass
```

**面试回答要点**：
- 优点：解耦、可测试、代码简洁
- 与 Spring Boot 的 @Autowired 类似
- 支持异步依赖（async def）

### 2. 知识点：SQLAlchemy ORM 关系映射

**原理**：`relationship()` 在 ORM 层建立 Python 对象间的导航属性，`back_populates` 实现双向引用。`ForeignKey` 在数据库层建立外键约束，`cascade="all, delete-orphan"` 确保父对象删除时子对象自动删除。

**底层实现**：
1. `relationship()` 创建一个描述符（descriptor）
2. 访问属性时触发延迟加载（lazy loading）
3. `back_populates` 建立双向同步机制
4. 级联操作通过数据库外键约束 + ORM 事件系统实现

**应用体现**：
```python
# User 模型
class User(Base):
    pets = relationship("Pet", back_populates="user", cascade="all, delete-orphan")

# Pet 模型
class Pet(Base):
    user_id = Column(ForeignKey("users.user_id", ondelete="CASCADE"))
    user = relationship("User", back_populates="pets")

# 删除用户时，其所有宠物自动删除
db.delete(user)
db.commit()  # pets 表中的相关记录也被删除
```

**面试回答要点**：
- lazy loading vs eager loading
- N+1 查询问题及解决方案（joinedload）
- 级联删除的数据库层面 vs ORM 层面

### 3. 知识点：Pydantic v2 数据验证

**原理**：Pydantic v2 使用 Rust 编写的核心（pydantic-core）实现高性能数据验证。`@field_validator` 在字段级别添加自定义验证逻辑，`model_dump(exclude_unset=True)` 只导出用户显式设置的字段，实现 PATCH 语义的部分更新。

**底层实现**：
1. 模型定义时，Pydantic 分析字段类型和约束
2. 实例化时，调用 Rust 核心进行验证
3. `@field_validator` 注册到验证器链
4. `exclude_unset=True` 通过 `__fields_set__` 属性跟踪已设置字段

**应用体现**：
```python
class PetUpdate(BaseModel):
    name: Optional[str] = None
    species: Optional[str] = None
    # 所有字段为 Optional
    
    @field_validator("species")
    def validate_species(cls, v):
        if v and v.lower() not in ALLOWED_SPECIES:
            raise ValueError("无效物种")
        return v.lower() if v else v

# 部分更新：只更新用户提交的字段
update_data = PetUpdate(name="新名字")  # species 未设置
pet_data = update_data.model_dump(exclude_unset=True)
# pet_data = {"name": "新名字"}  # 不包含 species
```

**面试回答要点**：
- Pydantic v2 比 v1 快 5-50 倍
- model_dump vs dict() 的区别
- Field() 函数的约束参数
