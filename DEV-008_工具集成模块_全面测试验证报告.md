### 3.2 失败的测试（3个）⚠️

#### 失败项 1：Pydantic 配置验证

**测试名称**: `test_config_model_validation`

**失败原因**: 
```
Failed: DID NOT RAISE <class 'pydantic_core._pydantic_core.ValidationError'>
```

**分析**: Pydantic V2 的验证行为变化。在 Pydantic V2 中，`storage_provider="minio"` 是有效值（因为枚举值未严格限制），不再抛出 ValidationError。

**修复建议**: 更新测试用例以匹配 Pydantic V2 的行为，或使用更严格的字段验证。

**影响程度**: 🟡 **低** - 仅影响测试逻辑，不影响实际功能

---

#### 失败项 2 & 3：Celery 模块缺失

**测试名称**: 
- `test_service_creation`
- `test_service_available_after_init_mock`

**失败原因**: 
```
ModuleNotFoundError: No module named 'celery'
```

**分析**: Celery 库未安装在当前环境中。

**修复方案**:
```bash
pip install celery[redis] redis
```

**影响程度**: 🟡 **中** - 异步任务服务需要安装依赖才能使用
