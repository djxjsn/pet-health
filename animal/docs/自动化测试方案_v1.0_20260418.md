## 一、方案概述

### 1.1 方案目标

建立一套覆盖 **单元测试 → 集成测试 → 接口测试 → UI 测试** 的全链路自动化测试体系，确保：

| 目标 | 指标 |
|------|------|
| 核心业务代码覆盖率 | ≥ 80%（目标 90%） |
| API 端点覆盖率 | 100%（50+ 端点） |
| 测试执行时间 | 单元+集成 ≤ 3 分钟，全量 ≤ 10 分钟 |
| 缺陷检出率 | P0/P1 级缺陷 100% 覆盖 |
| CI/CD 集成 | 提交即触发，阻塞合并 |

### 1.2 测试金字塔

```
              ╱╲
             ╱ E2E ╲          ← UI/E2E 测试 (少量，关键路径)
            ╱────────╲
           ╱  API 测试  ╲      ← 接口自动化 (中等，全部端点)
          ╱──────────────╲
         ╱   集成测试     ╲    ← 模块交互 (较多，核心流程)
        ╱──────────────────╲
       ╱     单元测试       ╲  ← 函数/方法级 (最多，基础保障)
      ╱──────────────────────╲
```

### 1.3 方案范围

| 层级 | 测试对象 | 数量预估 |
|------|---------|----------|
| L4 - UI/E2E | 前端页面交互流程 | ~30 用例 |
| L3 - API | 后端 RESTful 端点 | ~200 用例 |
| L2 - 集成 | 模块间协作、数据库操作 | ~150 用例 |
| L1 - 单元 | 函数/类/方法级别 | ~600 用例 |
| **合计** | | **~980 用例** |

### 4.3 单元测试用例设计标准

```python
# 用例命名规范: test_{功能}_{场景}_{预期结果}
class TestEncryptionService:
    
    def test_encrypt_valid_plaintext_returns_base64_ciphertext(self):
        """正常加密: 有效明文应返回Base64编码的密文"""
        
    def test_decrypt_tampered_ciphertext_raises_exception(self):
        """篡改检测: 被篡改的密文应抛出异常"""
        
    def test_encrypt_empty_string_returns_non_empty_result(self):
        """边界值: 空字符串加密应返回非空结果"""
        
    def test_encrypt_unicode_chinese_characters(self):
        """国际化: 中文/特殊字符加密正确性"""
        
    @pytest.mark.parametrize("plaintext", ["short", "a"*10000, ""])
    def test_encrypt_various_lengths(self, plaintext):
        """参数化: 不同长度文本加密"""
```

**设计原则**:
1. **AAA 模式**: Arrange → Act → Assert
2. **单一职责**: 每个测试只验证一个行为
3. **边界值**: 必须覆盖空值、最大值、非法值
4. **参数化**: 使用 `@pytest.mark.parametrize` 减少重复
5. **独立性**: 测试之间无依赖，可任意顺序执行
6. **自描述**: 函数名和 docstring 清晰说明测试意图

### 10.3 自定义测试摘要脚本

```python
# scripts/generate_test_summary.py
"""生成综合测试摘要报告"""
import json
from pathlib import Path
from datetime import datetime

def generate_summary():
    summary = {
        "timestamp": datetime.now().isoformat(),
        "project": "AI Pet Health Assistant",
        "version": "v9.0",
        "sections": {
            "unit_tests": _read_coverage_json(),
            "api_tests": _read_allure_results(),
            "e2e_tests": _read_playwright_results(),
            "frontend_unit": _read_jest_coverage(),
        }
    }
    
    # 计算总体指标
    total_cases = sum(s["total"] for s in summary["sections"].values())
    total_passed = sum(s["passed"] for s in summary["sections"].values())
    summary["overall"] = {
        "total_cases": total_cases,
        "passed": total_passed,
        "failed": total_cases - total_passed,
        "pass_rate": f"{total_passed/total_cases*100:.1f}%",
        "coverage": _calculate_weighted_coverage(summary["sections"]),
        "status": "PASS" if total_passed == total_cases else "FAIL"
    }
    
    Path("reports/summary/latest.json").write_text(
        json.dumps(summary, indent=2, ensure_ascii=False)
    )
    return summary
```

## 十一、缺陷管理流程

### 11.1 缺陷等级定义

| 等级 | 定义 | 响应时间 | 修复时限 | 示例 |
|------|------|---------|---------|------|
| **P0 - 致命** | 系统崩溃/数据丢失/安全漏洞 | 立即 | 24h | JWT绕过/数据库泄露 |
| **P1 - 严重** | 核心功能不可用/数据错误 | 4h | 72h | 登录失败/宠物创建失败 |
| **P2 - 一般** | 非核心功能异常/UI问题 | 1天 | 1周 | 排序不正确/样式错乱 |
| **P3 - 轻微** | 体验优化/建议改进 | 3天 | 2周 | 提示文案优化 |

### 11.2 缺陷生命周期

```
新建(New) 
  → 确认确认(Confirmed) 
    → 处理中(In Progress) 
      → 已修复(Fixed) 
        → 验证中(Verification) 
          → 已关闭(Closed)
          → 重新打开(Reopened) → 处理中
    → 拒绝/延后(Rejected/Deferred) → 已关闭
  → 无法复现(Cannot Reproduce) → 需更多信息
```

## 十二、CI/CD 集成方案

### 12.1 CI 流水线架构

```
Push/PR 触发
    │
    ├─► Stage 1: Lint & Format Check (≤ 2min)
    │   ├─ ruff check (Python)
    │   ├─ mypy type check (Python)
    │   └─ eslint + prettier (Frontend)
    │
    ├─► Stage 2: Unit Tests (≤ 3min) ⬅️ 阻塞合并
    │   ├─ pytest tests/unit/ --cov=src
    │   ├─ jest --coverage (Frontend)
    │   └─ 覆盖率门禁检查 (≥80%)
    │
    ├─► Stage 3: Integration Tests (≤ 3min) ⬅️ 阻塞合并
    │   ├─ pytest tests/integration/
    │   └─ pytest tests/api/
    │
    ├─► Stage 4: E2E Tests (≤ 10min) ⬅️ 不阻塞合并
    │   └─ playwright test
    │
    ├─► Stage 5: Reports Generation
    │   ├─ Coverage HTML Report
    │   ├─ Allure API Report
    │   ├─ Playwright E2E Report
    │   └─ Summary Report
    │
    └─► Stage 6: Notify
        ├─ Slack/DingTalk 通知
        └─ Email 发送报告
```

on:

    branches: [main]

      - uses: actions/setup-python@v5

        run: |
          cd animal
          pytest tests/ -v --cov=src --cov-report=xml --cov-report=html \
            --cov-fail-under=80 --junitxml=junit-unit.xml
      - name: Upload coverage report
        uses: actions/upload-artifact@v4
        with:
          name: coverage-report
          path: animal/coverage_html_report/

      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5

        with:
          results: animal/allure-results
      - name: Upload Allure report
        uses: actions/upload-artifact@v4
        with:
          name: allure-report
          path: animal/allure-report/

  frontend-tests:
    name: 🎨 Frontend Tests
    runs-on: ubuntu-latest
    needs: lint
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
          cache-dependency-path: frontend/package-lock.json
      - name: Install and test
        run: |
          cd frontend
          npm ci
          npm run test:coverage
      - name: Upload frontend coverage
        uses: actions/upload-artifact@v4
        with:
          name: frontend-coverage
          path: frontend/coverage/

  e2e-tests:
    name: 🌐 E2E Tests
    runs-on: ubuntu-latest
    needs: integration-api-tests
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - name: Install dependencies
        run: cd frontend && npm ci && npx playwright install --with-deps chromium
      - name: Run E2E tests
        run: cd frontend && npx playwright test
      - name: Upload E2E report
        uses: actions/upload-artifact@v4
        if: always()
        with:
          name: playwright-report
          path: frontend/playwright-report/

  notify:
    name: 📢 Notify
    runs-on: ubuntu-latest
    needs: [unit-tests, integration-api-tests, frontend-tests, e2e-tests]
