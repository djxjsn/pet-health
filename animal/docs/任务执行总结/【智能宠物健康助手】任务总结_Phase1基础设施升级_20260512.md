## 遇到的问题

1. **Qdrant PointStruct稀疏向量格式问题**: 初始使用dict传入稀疏向量，Qdrant要求使用SparseVector对象(indices+values)，导致ValidationError
2. **_encode方法兼容性**: mock测试中encode返回list而非numpy array，调用.tolist()报AttributeError
3. **旧测试import路径**: 旧test_vector_db.py引用`from src.core.vector_db import VectorDatabase`，重构后路径变更

## 解决方案

1. 在add_documents中将稀疏向量dict转换为`SparseVector(indices=..., values=...)`对象
2. 在_encode方法中添加hasattr检查，兼容numpy array和原生list
3. 更新旧测试import为`from src.core.vector_db_chroma import VectorDatabase`，patch路径同步更新

## 经验与改进建议

1. **Qdrant稀疏向量必须使用SparseVector对象**: 不能直接传dict，这是Qdrant客户端的硬性要求
2. **接口兼容性设计很重要**: get_vector_db()路由函数使得所有调用方无需修改，降低了迁移风险
3. **测试先行发现集成问题**: mock测试在早期就发现了SparseVector格式问题，避免了运行时错误
4. **渐进式迁移策略有效**: 保留ChromaDB兼容，通过配置切换，降低了迁移风险
