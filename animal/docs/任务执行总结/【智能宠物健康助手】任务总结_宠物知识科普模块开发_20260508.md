## 遇到的问题

### 1. Next.js 动态路由冲突（[species] vs [breedId]）
- **现象**：`/knowledge/breeds/[species]` 和 `/knowledge/breeds/[breedId]` 使用相同模式
- **分析**：Next.js App Router中，同一路径段不能有两个不同的动态参数
- **解决**：列表页改用 query params（`/knowledge/breeds?species=cat`），详情页保留动态路由（`/knowledge/breeds/[breedId]`）
- **影响范围**：同样修复了 health 路由

### 2. Tailwind CSS 颜色系统不兼容
- **现象**：使用 `bg-green-50`、`text-amber-500` 等内置颜色类时报错
- **分析**：项目 `tailwind.config.js` 在 `theme.colors` 层级（非 `extend`）重定义了全部颜色，覆盖了默认调色板
- **解决**：全量映射：green→success, amber→secondary, red/rose→primary, blue→accent, emerald→success, violet→primary
- **影响范围**：5个页面文件，共12处替换

### 3. ApiResponse 类型导入错误
- **现象**：编译报错 `Module '"./client"' declares 'ApiResponse' locally, but it is not exported`
- **分析**：encyclopedia.ts 从 `./client` 导入 `ApiResponse`，但 client.ts 只内部使用未重新导出
- **解决**：改为从 `@/types/api` 直接导入

### 4. apiClient.get() 参数类型错误
- **现象**：`{ params: { query } }` 报类型不匹配
- **分析**：`apiClient.get<T>(endpoint, params)` 第二个参数为 `Record<string, string | number | boolean>`，不是配置对象
- **解决**：直接传入 `{ query }`

---

## 解决方案

所有问题通过以下方式解决：
1. **路由问题**：采用"列表用 query params + 详情用动态路由"的双模式路由设计
2. **颜色问题**：建立颜色映射表，用正则扫描全模块统一修复
3. **类型问题**：追踪 import 链，定位正确的导出源
4. **参数问题**：阅读 ApiClient 源码签名，调整传参方式

## 经验与改进建议

### 经验
1. **先读源码，后写代码**：Tailwind 颜色问题如果能先检查 tailwind.config.js 可避免后续返工
2. **路由设计前置**：涉及列表+详情的模块，应在一开始就确定 query params + dynamic route 的组合策略
3. **自动颜色检查**：可以编写一个 lint 脚本自动检测项目中是否使用了非项目色系的颜色类

### 改进建议

2. 可添加图片资源支持（当前使用 emoji 占位）
