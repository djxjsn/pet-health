### ✅ 步骤 3: 用户认证 API 实现

**实现的 API 端点**:

#### 认证端点 (`/api/v1/auth`)

1. **用户注册** - `POST /auth/register`
   - 验证手机号和邮箱格式
   - 检查手机号和邮箱是否重复
   - 密码强度验证（最少 8 个字符，包含字母和数字）
   - 自动创建用户档案

2. **用户登录** - `POST /auth/login`
   - 支持手机号或邮箱登录
   - 验证用户凭证
   - 返回 access_token 和 refresh_token
   - 更新最后登录时间

3. **令牌刷新** - `POST /auth/refresh`
   - 验证 refresh_token 有效性
   - 生成新的 access_token 和 refresh_token
   - 防止令牌伪造

4. **忘记密码** - `POST /auth/forgot-password`
   - 验证用户是否存在
   - 生成密码重置令牌（6 小时有效期）
   - TODO: 发送邮件功能

5. **重置密码** - `POST /auth/reset-password`
   - 验证重置令牌
   - 验证新密码强度
   - 更新用户密码
   - TODO: 完整的令牌验证逻辑

#### 用户管理端点 (`/api/v1/users`)

1. **获取当前用户信息** - `GET /users/me`
   - 需要认证
   - 返回用户基本信息和档案

2. **更新当前用户信息** - `PUT /users/me`
   - 需要认证
   - 可更新昵称、头像、手机号
   - 验证手机号和邮箱唯一性

3. **更新当前用户档案** - `PUT /users/me/profile`
   - 需要认证
   - 可更新宠物数量、地址等档案信息

**安全特性**:
- 密码使用 bcrypt 哈希
- JWT 令牌认证（HS256 算法）
- Access token 有效期 30 分钟
- Refresh token 有效期 7 天
- OAuth2 Password Bearer 认证流程
- 依赖注入确保端点安全
