## 🛠️ 遇到的问题与解决方案

### 问题 1: Pydantic 配置验证错误

**问题描述**:  
测试运行时出现 `ValidationError: Extra inputs are not permitted` 错误

**根本原因**:  
`.env` 文件中的 `OPENAI_API_KEY` 和 `DEEPSEEK_API_KEY` 占位值不被 Pydantic 允许

**解决方案**:  
修改 `src/core/config.py`:
- 将 API key 字段改为 `Optional[str] = None`
- 添加 `"extra": "ignore"` 配置项

**结果**: ✅ 问题解决，测试正常运行

---

### 问题 2: 数据库连接 Unicode 编码错误

**问题描述**:  
`UnicodeEncodeError: 'latin-1' codec can't encode characters`

**根本原因**:  
数据库密码包含中文字符（"你的数据库密码"），PyMySQL 无法编码

**解决方案**:  
1. 修改 `.env` 文件，使用英文占位符密码
2. 添加注释提示用户使用英文密码
3. 创建 `.env.local` 模板文件

**结果**: ✅ 服务器可以正常启动（需用户配置正确密码）

---

### 问题 3: 测试环境数据库隔离

**问题描述**:  
测试时尝试连接真实 MySQL 数据库，导致测试不独立

**解决方案**:  
1. 在 `conftest.py` 中设置环境变量覆盖
2. 使用 SQLite 内存数据库进行测试
3. 修改 `database.py` 支持 SQLite 配置
4. 添加测试环境检测，跳过启动时的表创建

**结果**: ✅ 测试完全隔离，不影响生产数据

---

### 问题 4: 缺少导入函数

**问题描述**:  
`NameError: name 'get_user_by_id' is not defined`

**根本原因**:  
`auth.py` 中使用了未导入的 CRUD 函数

**解决方案**:  
在导入语句中添加 `get_user_by_id`

**结果**: ✅ 令牌刷新测试通过

```bash

..\.venv\Scripts\Activate.ps1

# 2. 修改 .env 文件中的数据库密码

```

### 访问 API 文档

- **Swagger UI**: http://localhost:8000/docs
- **ReDoc**: http://localhost:8000/redoc

```bash

python -m pytest tests/ -v

python -m pytest tests/ -v --cov=src --cov-report=term-missing
```
