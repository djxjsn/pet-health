## 📝 网页版使用步骤

### 步骤 1: 注册/登录账号

1. 访问 https://www.apifox.cn/
2. 点击右上角「登录/注册」
3. 选择登录方式：
   - 手机号注册（推荐）
   - 微信扫描登录
   - GitHub 账号登录
   - 企业账号登录

### 步骤 2: 创建项目

1. 登录后进入控制台
2. 点击「新建项目」
3. 填写项目信息：
   ```
   项目名称：AI 宠物健康助手
   项目描述：基于 LangChain 的 AI 宠物健康助手 API
   项目标签：宠物健康、AI、FastAPI
   ```
4. 点击「创建」

### 步骤 3: 导入接口文档

#### 方法 A: URL 导入（推荐）
```
1. 启动 FastAPI 服务
   cd animal
   uvicorn src.main:app --reload

2. 在网页版中：
   项目 → 接口 → 导入 → URL
   输入：http://localhost:8000/openapi.json
   
   注意：如果是本地服务，需要使用客户端版或确保网络可达
```

#### 方法 B: 文件导入（推荐网页版用户）
```
1. 导出 OpenAPI 文档：
   浏览器访问：http://localhost:8000/openapi.json
   保存为 openapi.json

2. 在网页版中：
   项目 → 接口 → 导入 → OpenAPI/Swagger
   选择 openapi.json 文件
   点击「导入」
```

#### 方法 C: 手动创建（适合少量接口）
```
1. 项目 → 接口 → 新建接口
2. 填写接口信息
3. 保存
```

### 步骤 4: 配置环境

```
1. 项目设置 → 环境管理 → 新增环境

开发环境配置:
  环境名称：开发环境
  前置域名：http://localhost:8000
  全局变量：
    - base_url: http://localhost:8000/api/v1
    - access_token: (留空，自动获取)
```

### 步骤 5: 设置全局认证

```
1. 项目设置 → 认证设置 → Token 认证
2. 配置：
   - Token 名称：access_token
   - 应用范围：全局生效
3. 保存
```

### 步骤 6: 创建测试用例

#### 创建测试集合
```
1. 测试 → 测试集合 → 新建测试集合
2. 命名：AI 对话功能完整测试
3. 从接口列表拖拽接口到测试集合
```

#### 配置测试数据
```
1. 点击接口进入编辑
2. 在「测试数据」标签页添加用例
3. 配置请求参数
4. 添加验证规则
```

### 步骤 7: 运行测试

```
1. 进入测试集合
2. 选择环境（开发环境）
3. 点击「运行」
4. 查看实时结果
```

## 🎯 网页版测试我们的项目

### 完整测试流程

#### 1. 准备阶段
```bash
# 启动 FastAPI 服务
cd animal
uvicorn src.main:app --reload --host 0.0.0.0 --port 8000
```

#### 2. 导出接口文档
```
浏览器访问：http://localhost:8000/openapi.json
保存为：openapi.json
```

#### 3. 网页版导入
```
登录 apifox.cn → 选择项目 → 接口 → 导入 → OpenAPI
选择 openapi.json → 导入成功
```

#### 4. 配置测试数据
```
前置脚本（项目级）:
// 自动登录获取 Token
const loginReq = {
  url: "http://localhost:8000/api/v1/auth/login",
  method: "POST",
  body: "username=13800000001&password=test1234"
};

pm.sendRequest(loginReq, (err, res) => {
  const token = res.json().access_token;
  pm.environment.set("access_token", token);
});
```

#### 5. 运行测试
```
测试 → 选择测试集合 → 运行
查看实时结果和报告
```

## ⚠️ 网页版注意事项

### 1. 网络要求
```
- 需要稳定的网络连接
- 本地服务需要确保网络可达
- 建议使用客户端版测试纯本地服务
```

### 2. 浏览器兼容性
```
推荐浏览器：
- Chrome 90+
- Edge 90+
- Firefox 88+
- Safari 14+
```

### 3. 性能优化
```
- 接口数量多时，使用搜索功能
- 定期清理历史运行记录
- 大项目分模块管理
```

### 4. 安全建议
```
- 不要分享个人账号
- 敏感数据使用环境变量
- 生产环境 Token 定期更换
```

## 📞 常见问题

### Q1: 网页版能测试本地服务吗？
**A**: 可以，但需要确保：
- 服务监听 0.0.0.0 而非 localhost
- 防火墙允许外部访问
- 或使用 ngrok 等工具暴露内网

**推荐方案**: 本地开发用客户端，测试环境用网页版

### Q2: 网页版数据会丢失吗？
**A**: 不会，APIFox 提供：
- 实时云端存储
- 自动备份
- 版本历史
- 导出功能

### Q3: 网页版免费吗？
**A**: 是的，提供：
- 免费版：个人和小团队足够
- 付费版：高级功能和更大容量

### Q4: 网页版支持 WebSocket 吗？
**A**: 支持！现代浏览器都支持 WebSocket，可以直接测试我们的 /ws/chat 接口。
