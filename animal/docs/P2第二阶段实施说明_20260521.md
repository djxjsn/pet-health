# P2 第二阶段实施说明（2026-05-21）

## 目标

在 P2 第一阶段基础上，将质量门禁能力扩展到症状分析与营养建议工具路径，减少“主链路外”工具直接返回低可信建议的风险。

## 本次实施内容

### 1) `analyze_symptoms` 接入质量门禁

- 文件：`animal/src/tools/pet_health_tools.py`
- 变更点：
  - 从 `search_for_symptom_analysis(...)` 切换为 `search_with_quality(...)`
  - 增加输出字段：
    - `quality_action`
    - `quality_confidence`
  - `action=refuse` 时推荐语句升级为强转诊导向

### 2) `get_nutrition_advice` 接入质量门禁

- 文件：`animal/src/tools/pet_health_tools.py`
- 变更点：
  - 构建营养检索 query 后统一走 `search_with_quality(...)`
  - 增加输出字段：
    - `quality_action`
    - `quality_confidence`
  - `action=refuse` 且无结果时，返回保守营养建议占位，避免空输出

### 3) 回归测试

- 新增：`animal/tests/test_p2_quality_gate_tools.py`
- 覆盖：
  - `SymptomAnalysisTool` 是否调用 `search_with_quality`
  - `NutritionAdviceTool` 在 `refuse` 下是否给出保守结果

## 验证命令

```bash
python -m pytest "animal/tests/test_p2_quality_gate_tools.py" -q
```
