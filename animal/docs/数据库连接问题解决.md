# 🔧 数据库连接问题解决方案

## ❌ 当前错误

```
sqlalchemy.exc.OperationalError: (pymysql.err.OperationalError) (1045, "Access denied for user 'root'@'localhost' (using password: YES)")
```

**原因**: 数据库密码不正确

---

## ✅ 解决方案（3 选 1）

### 方案 1: 修改 .env 文件（推荐）

#### 步骤 1: 找到你的 MySQL 密码
```
如果你记得密码，直接跳到步骤 2

如果不记得密码，需要重置 MySQL 密码（见方案 2）
```

#### 步骤 2: 编辑 .env 文件
```
文件位置：animal/.env
找到第 11 行：
DATABASE_URL=mysql+pymysql://root:test_password@localhost:3306/ai_pet_health

将 test_password 替换为你的实际密码
例如：DATABASE_URL=mysql+pymysql://root:123456@localhost:3306/ai_pet_health
```

#### 步骤 3: 重启服务
```bash
# 停止当前服务（按 Ctrl+C）
# 然后重新启动
cd animal
python -m uvicorn src.main:app --reload
```

---

### 方案 2: 重置 MySQL 密码（如果忘记密码）

#### Windows 系统

**步骤 1: 停止 MySQL 服务**
```
方法 A: 使用服务管理器
  Win+R → services.msc → 找到 MySQL → 右键停止

方法 B: 使用命令行（管理员）
  net stop MySQL
```

**步骤 2: 跳过密码验证启动**
```bash
# 找到 MySQL 安装目录
# 通常在 C:\Program Files\MySQL\MySQL Server X.X\bin\

# 以跳过权限表模式启动
mysqld --skip-grant-tables
```

**步骤 3: 重置密码**
```bash
# 打开新的命令行窗口
mysql -u root

# 执行 SQL 命令
USE mysql;
ALTER USER 'root'@'localhost' IDENTIFIED BY '新密码';
FLUSH PRIVILEGES;
EXIT;
```

**步骤 4: 重启 MySQL**
```
1. 关闭 --skip-grant-tables 窗口
2. 在服务管理器中启动 MySQL
3. 或使用：net start MySQL
```

**步骤 5: 更新 .env 文件**
```
编辑 animal/.env
DATABASE_URL=mysql+pymysql://root:新密码@localhost:3306/ai_pet_health
```

---

### 方案 3: 使用其他数据库用户

#### 创建新用户
```sql
-- 登录 MySQL
mysql -u root -p

-- 创建新用户
CREATE USER 'pet_user'@'localhost' IDENTIFIED BY 'pet_password';

-- 授权
GRANT ALL PRIVILEGES ON ai_pet_health.* TO 'pet_user'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

#### 更新 .env 文件
```
DATABASE_URL=mysql+pymysql://pet_user:pet_password@localhost:3306/ai_pet_health
```

### 方法 2: 使用 Python 脚本
```python
# 创建 test_db.py
import pymysql

try:
    connection = pymysql.connect(
        host='localhost',
        user='root',
        password='你的密码',  # 替换为实际密码
        database='ai_pet_health'
    )
    print("✓ 数据库连接成功！")
    connection.close()
except Exception as e:
    print(f"✗ 连接失败：{e}")
```

## 📞 常见问题

### Q1: 数据库不存在怎么办？
**A**: 创建数据库
```sql
CREATE DATABASE ai_pet_health DEFAULT CHARACTER SET utf8mb4;
```

### Q2: 提示"表不存在"？
**A**: 运行数据库迁移
```bash
cd animal
alembic upgrade head
```

### Q3: 还是连接失败？
**A**: 检查以下几点
1. MySQL 服务是否启动？
2. 端口是否正确（默认 3306）？
3. 防火墙是否阻止？
4. 用户权限是否正确？

### Q4: 使用其他端口？
**A**: 修改 .env 文件
```
DATABASE_URL=mysql+pymysql://root:密码@localhost:3307/ai_pet_health
#                                                            ^^^^
#                                                            修改端口
```
