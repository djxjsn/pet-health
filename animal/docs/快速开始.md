# 快速开始指南

## 开发服务器启动

### 前提条件

1. **Python 虚拟环境已激活**
   ```bash
   # Windows PowerShell
   ..\.venv\Scripts\Activate.ps1
   
   # 或者使用命令
   source .venv/Scripts/activate
   ```

2. **数据库已配置**
   - MySQL 服务正在运行
   - 数据库 `ai_pet_health` 已创建
   - 已设置 root 用户密码

### 步骤 1: 配置数据库密码

编辑 `.env` 文件，将数据库密码替换为你的实际密码：

```bash
# 打开 .env 文件
DATABASE_URL=mysql+pymysql://root:YOUR_PASSWORD@localhost:3306/ai_pet_health
```

**重要**: 将 `YOUR_PASSWORD` 替换为你的 MySQL root 密码（不能使用中文密码）。

### 步骤 2: 启动开发服务器

#### 方式一：使用启动脚本（推荐）

```bash
# 进入项目目录
cd animal

# 启动开发服务器
python scripts/dev_server.py
```

**可选参数**:
```bash
# 自定义端口
python scripts/dev_server.py --port 8080

# 禁用热重载
python scripts/dev_server.py --no-reload

# 查看帮助
python scripts/dev_server.py --help
```

#### 方式二：直接使用 uvicorn

```bash
uvicorn src.main:app --reload --host 0.0.0.0 --port 8000
```

### 步骤 3: 验证服务器运行

服务器启动后，你会看到类似输出：

```
INFO:     Uvicorn running on http://0.0.0.0:8000 (Press CTRL+C to quit)
INFO:     Started reloader process [12345] using WatchFiles
INFO:     Started server process [67890]
INFO:     Waiting for application startup.
INFO:     Application startup complete.
```

### 步骤 4: 访问 API 文档

打开浏览器访问：

- **Swagger UI**: http://localhost:8000/docs
- **ReDoc**: http://localhost:8000/redoc
- **健康检查**: http://localhost:8000/health

### 步骤 5: 测试 API 端点

#### 使用 Swagger UI 测试

1. 访问 http://localhost:8000/docs
2. 展开 `/api/v1/auth/register` 端点
3. 点击 "Try it out"
4. 填写注册信息：
   ```json
   {
     "phone": "13800138000",
     "email": "test@example.com",
     "password": "test1234"
   }
   ```
5. 点击 "Execute" 执行请求

#### 使用 curl 测试

```bash
# 健康检查
curl http://localhost:8000/health

# 用户注册
curl -X POST "http://localhost:8000/api/v1/auth/register" \
  -H "Content-Type: application/json" \
  -d '{"phone":"13800138000","email":"test@example.com","password":"test1234"}'

# 用户登录
curl -X POST "http://localhost:8000/api/v1/auth/login" \
  -d "username=13800138000" \
  -d "password=test1234"
```

## 常见问题

### 问题 1: 数据库连接失败

**错误信息**:
```
pymysql.err.OperationalError: (1045, "Access denied for user 'root'@'localhost'")
```

**解决方案**:
1. 检查 `.env` 文件中的数据库密码是否正确
2. 确保 MySQL 服务正在运行
3. 验证数据库 `ai_pet_health` 已创建

### 问题 2: 端口已被占用

**错误信息**:
```
OSError: [Errno 48] Address already in use
```

**解决方案**:
```bash
# 使用其他端口
python scripts/dev_server.py --port 8080
```

或者关闭占用端口的进程。

### 问题 3: 虚拟环境未激活

**错误信息**:
```
ModuleNotFoundError: No module named 'fastapi'
```

**解决方案**:
```bash
# Windows PowerShell
..\.venv\Scripts\Activate.ps1

# 或者
.venv\Scripts\activate
```

### 问题 4: 密码包含中文字符

**错误信息**:
```
UnicodeEncodeError: 'latin-1' codec can't encode characters
```

**解决方案**:
- 数据库密码不能使用中文
- 修改 `.env` 文件，使用英文密码

## 开发技巧

### 1. 使用 Swagger UI 的 Authorize 功能

1. 登录获取 token
2. 点击页面右上角 "Authorize" 按钮
3. 输入 `Bearer <your_access_token>`
4. 点击 Authorize
5. 现在可以测试需要认证的端点

### 2. 查看实时日志

开发服务器会实时显示请求日志：

```
INFO:     127.0.0.1:12345 - "POST /api/v1/auth/register HTTP/1.1" 201 Created
INFO:     127.0.0.1:12346 - "POST /api/v1/auth/login HTTP/1.1" 200 OK
```

### 3. 启用调试模式

在 `.env` 文件中设置：

```bash
DEBUG=True
DATABASE_ECHO=True
```

这将显示详细的错误信息和 SQL 日志。
