# DEV-008 工具集成模块 - 功能说明文档

## 1. 模块概述

### 1.1 功能定位

DEV-008 工具集成模块是 AI 宠物健康助手的核心基础设施，提供：

- **内部工具集**：宠物健康、行为分析、营养建议等业务工具
- **外部 API 集成**：天气查询、地图服务、网络搜索、图像识别
- **工具调用框架**：统一调用、缓存、重试、并行执行、链式编排

### 1.2 模块架构

```
src/tools/
├── base.py                 # 基础工具类 (BaseTool)
├── tool_registry.py        # 工具注册管理器 (ToolRegistry)
├── tool_executor.py        # 工具调用执行器 (ToolExecutor)
├── pet_health_tools.py     # 宠物健康工具集 (5个)
├── health_tools.py         # 健康咨询工具集 (3个)
├── behavior_tools.py       # 行为分析工具集 (2个)
└── external_tools.py       # 外部API集成工具 (5个) [新增]
```

## 3. 使用示例

### 3.1 基础用法 - 单个工具调用

```python
from src.tools.tool_executor import get_tool_executor

# 创建执行器
executor = get_tool_executor()

# 调用天气工具
result = executor.call_tool("get_weather", city="北京", days=3)

if result.status.value == "success":
    print(f"当前温度: {result.result['current']['temp']}°C")
    for advice in result.result['pet_advice']:
        print(f"建议: {advice}")
```

### 3.2 并行调用多个工具

```python
# 同时调用多个工具
tool_calls = [
    {"tool_name": "get_weather", "city": "上海"},
    {"tool_name": "search_nearby", "location": "上海市浦东新区", "query_type": "hospital"},
    {"tool_name": "web_search", "query": "夏季宠物防暑指南"}
]

results = executor.call_tools_parallel(tool_calls)

for result in results:
    if result.status.value == "success":
        print(f"{result.tool_name}: 成功")
```

### 3.3 链式调用 - 复杂工作流

```python
# 定义工具链
chain = [
    # 步骤1: 获取天气
    {
        "tool_name": "get_weather",
        "city": "北京",
        "output_key": "weather_info"
    },
    # 步骤2: 根据天气搜索相关健康知识
    {
        "tool_name": "enhance_knowledge",
        "topic": "$weather_info",  # 引用上一步输出
        "depth": "deep"
    },
    # 步骤3: 搜索附近的宠物医院
    {
        "tool_name": "search_nearby",
        "location": "北京市朝阳区",
        "query_type": "hospital"
    }
]

result = executor.call_tool_chain(chain)

if result["success"]:
    print(f"完成 {len(result['steps'])} 个步骤")
    for step in result["steps"]:
        print(f"  {step['step']}. {step['tool_name']}: {step['status']}")
```

### 3.4 图像识别使用

## 4. 工具调用框架特性

### 4.1 智能缓存

- 自动缓存相同参数的调用结果
- 可配置缓存有效期
- 支持手动清除缓存

```python
# 启用缓存
executor.call_tool("get_weather", city="北京")  # 首次调用，执行实际逻辑
executor.call_tool("get_weather", city="北京")  # 第二次，返回缓存结果

# 清除缓存
executor.clear_cache()
```

### 4.2 自动重试

- 可配置最大重试次数和间隔
- 仅对临时性错误进行重试
- 记录每次尝试的日志

### 4.3 并发支持

- 使用线程池并行执行多个工具调用
- 自动限制并发数量（最多5个）
- 统一收集所有结果

### 4.4 调用统计

```python
stats = executor.get_stats()
print(stats)
# {'get_weather': {'total_calls': 10, 'success_count': 9, ...}}
```

### 4.5 历史记录

```python
history = executor.get_call_history(limit=20)
for record in history:
    print(f"{record.tool_name}: {record.status.value} ({record.execution_time_ms:.0f}ms)")
```

## 5. 外部 API 接入说明

### 5.1 当前实现状态

当前外部工具使用**模拟数据**，便于开发和测试。生产环境需要：

1. **天气 API**: 接入 OpenWeatherMap / 和风天气
2. **地图 API**: 接入 高德地图 / 百度地图 / Google Maps
3. **搜索 API**: 接入 Tavily / Bing Search / Google Custom Search
4. **图像识别**: 接入 百度AI / Azure Computer Vision / AWS Rekognition

### 5.2 快速接入示例

以接入真实天气 API 为例：

```python
# external_tools.py 中修改 _fetch_weather_data 方法

def _fetch_weather_data(self, city: str, days: int) -> Dict[str, Any]:
    import requests
    
    api_key = self.config.weather_api_key  # 从配置获取
    url = f"https://api.openweathermap.org/data/2.5/forecast"
    
    params = {
        "q": city,
        "appid": api_key,
        "units": "metric",
        "cnt": days * 8  # 每天8个数据点
    }
    
    response = requests.get(url, params=params, timeout=10)
    response.raise_for_status()
    
    data = response.json()
    return self._parse_openweather_response(data, days)
```

## 6. 扩展开发指南

### 6.1 添加新工具

1. 在对应文件中创建工具类

```python
class MyNewTool(BaseTool):
    name: str = "my_new_tool"
    description: str = "工具描述"
    args_schema: Type[BaseModel] = MyInputModel
    
    def _run(self, param1: str, param2: int = 10) -> Dict[str, Any]:
        # 实现工具逻辑
        return {"result": "data"}
```

2. 在 tool_registry.py 中注册

```python
self.register_tool("my_new_tool", MyNewTool)
```

### 6.2 工具输入规范

- 使用 Pydantic BaseModel 定义输入参数
- 所有字段必须有描述
- 设置合理的默认值和验证规则

```python
class MyInput(BaseModel):
    required_field: str = Field(..., description="必填字段说明")
    optional_field: int = Field(10, description="可选字段", ge=1, le=100)
```
