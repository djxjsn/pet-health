# P2 第一阶段实施说明（2026-05-21）

## 目标

在不大规模重构主链路的前提下，先将质量门禁与安全后置策略接入生产路径，降低高风险问答误导概率，并提供最小离线评测脚手架。

## 本次实施范围

### 1) 质量门禁接入工具主路径

- 文件：`animal/src/tools/pet_health_tools.py`
- 变更：`HealthKnowledgeTool._run()` 从 `retriever.search()` 切换为 `retriever.search_with_quality(...)`
- 默认开启：
  - `enable_self_rag=True`
  - `enable_crag=True`
- 行为：
  - `action=refuse` 时返回显式拒答占位结果（建议面诊）
  - 其他 action 时把 `quality_action/quality_confidence` 注入结果 metadata

### 2) AgentV2 高风险后置安全校验

- 文件：`animal/src/agents/agent_v2.py`
- 变更：
  - 在 `integrate_v2()` 最终输出前调用 `_apply_safety_postcheck()`
  - 高风险意图（`emergency_assess` / `symptom_check`）如果回复中缺少关键安全标记，则自动追加强提醒与转诊建议

### 3) 最小离线评测脚手架

- 新增文件：`animal/scripts/run_p2_smoke_eval.py`
- 功能：
  - `eval_quality_gate()`：抽样检查 `search_with_quality` 的 action/confidence
  - `eval_safety_postcheck()`：验证高风险回复会自动追加安全后缀

## 运行方式

在项目根目录执行：

```bash
python animal/scripts/run_p2_smoke_eval.py
```

## 当前限制与后续建议

1. 当前为第一阶段接入，质量门禁优先接到了 `search_health_knowledge` 工具路径；后续建议把其他检索工具（症状/营养）也统一切换到同类门禁策略。
2. 安全后置校验目前基于关键词规则，后续可升级为结构化安全评分（risk score）并配合拒答模板统一输出。
3. 建议下一阶段补充：
   - Gold set（30~50条）与批量离线评测报告
   - 线上指标面板（quality action 分布、refuse 率、紧急转诊率）
