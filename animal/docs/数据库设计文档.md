## 二、表结构设计

### 2.1 用户表 (users)

**表说明**: 存储用户的基本认证信息

**表结构**:
```sql
CREATE TABLE users (
    user_id CHAR(36) NOT NULL COMMENT '用户唯一标识符(UUID)',
    phone VARCHAR(20) DEFAULT NULL COMMENT '手机号(唯一)',
    email VARCHAR(255) DEFAULT NULL COMMENT '邮箱(唯一)',
    password_hash VARCHAR(255) NOT NULL COMMENT '密码哈希值',
    is_active TINYINT(1) NOT NULL DEFAULT 1 COMMENT '账户是否激活: 0-否, 1-是',
    is_superuser TINYINT(1) NOT NULL DEFAULT 0 COMMENT '是否超级管理员: 0-否, 1-是',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    last_login_at DATETIME DEFAULT NULL COMMENT '最后登录时间',
    PRIMARY KEY (user_id),
    UNIQUE KEY uk_phone (phone),
    UNIQUE KEY uk_email (email),
    INDEX idx_created_at (created_at),
    INDEX idx_is_active (is_active)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户表';
```

**字段说明**:

| 字段名 | 数据类型 | 约束 | 说明 |
|--------|----------|------|------|
| user_id | CHAR(36) | PK, NOT NULL | UUID 主键，程序生成 |
| phone | VARCHAR(20) | UNIQUE, NULL | 手机号，可为空 |
| email | VARCHAR(255) | UNIQUE, NULL | 邮箱，可为空 |
| password_hash | VARCHAR(255) | NOT NULL | bcrypt 哈希后的密码 |
| is_active | TINYINT(1) | NOT NULL, DEFAULT 1 | 账户激活状态 |
| is_superuser | TINYINT(1) | NOT NULL, DEFAULT 0 | 超级管理员标识 |
| created_at | DATETIME | NOT NULL | 账户创建时间 |
| updated_at | DATETIME | NOT NULL | 最后更新时间 |
| last_login_at | DATETIME | NULL | 最后登录时间 |

**设计说明**:
- `user_id` 使用 UUID 而不是自增 ID，便于分布式环境和数据迁移
- `phone` 和 `email` 至少有一个非空，实现双因子登录
- `password_hash` 使用 bcrypt 哈希，不存储明文密码
- `is_active` 支持账户禁用功能
- `is_superuser` 预留管理员权限字段

### 2.2 用户档案表 (user_profiles)

**表说明**: 存储用户的详细档案信息，与用户表一一对应

**表结构**:
```sql
CREATE TABLE user_profiles (
    user_id CHAR(36) NOT NULL COMMENT '用户ID(外键)',
    nickname VARCHAR(50) DEFAULT NULL COMMENT '用户昵称',
    avatar_url VARCHAR(500) DEFAULT NULL COMMENT '头像URL',
    pet_count INT NOT NULL DEFAULT 0 COMMENT '宠物数量',
    address VARCHAR(255) DEFAULT NULL COMMENT '地址',
    bio TEXT DEFAULT NULL COMMENT '个人简介',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    PRIMARY KEY (user_id),
    CONSTRAINT fk_profile_user FOREIGN KEY (user_id) REFERENCES users(user_id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户档案表';
```

**字段说明**:

| 字段名 | 数据类型 | 约束 | 说明 |
|--------|----------|------|------|
| user_id | CHAR(36) | PK, FK, NOT NULL | 关联 users 表的用户ID |
| nickname | VARCHAR(50) | NULL | 用户昵称 |
| avatar_url | VARCHAR(500) | NULL | 头像 URL 地址 |
| pet_count | INT | NOT NULL, DEFAULT 0 | 宠物数量 |
| address | VARCHAR(255) | NULL | 用户地址 |
| bio | TEXT | NULL | 个人简介/签名 |
| created_at | DATETIME | NOT NULL | 档案创建时间 |
| updated_at | DATETIME | NOT NULL | 最后更新时间 |

**设计说明**:
- 与 users 表一对一关系，通过 user_id 级联删除
- 将用户信息分为主表（认证）和从表（档案），符合职责分离原则
- `pet_count` 冗余字段，避免每次查询都关联宠物表统计
- `avatar_url` 支持外部存储的头像 URL

## 五、约束设计

### 5.1 用户表约束

1. **手机号唯一约束**: 同一手机号不能重复注册
2. **邮箱唯一约束**: 同一邮箱不能重复注册
3. **密码非空约束**: 密码哈希值不能为空
4. **活跃状态默认值**: 新用户默认激活状态

### 5.2 用户档案表约束

1. **外键约束**: user_id 引用 users 表的 user_id
2. **级联删除**: 删除用户时自动删除对应档案
3. **宠物数量非负**: pet_count >= 0

## 六、安全设计

### 6.1 密码安全

- **哈希算法**: bcrypt（cost factor = 12）
- **盐值**: 自动生成，每个密码独立盐值
- **验证方式**: 恒定时间比较，防止时序攻击

### 6.2 数据加密

- **敏感字段**: password_hash 加密存储
- **传输安全**: HTTPS 传输（生产环境）
- **日志脱敏**: 日志中不记录明文密码

### 6.3 访问控制

- **认证机制**: JWT Bearer Token
- **会话管理**: 令牌过期自动失效
- **权限验证**: 超级管理员标识控制管理权限

CREATE TABLE pets (
