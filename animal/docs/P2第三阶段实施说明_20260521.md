# P2 第三阶段实施说明（质量动作统计 + 自动评测报告）

## 目标

- 将 `quality_action` 维度并入 `/api/v1/chat/orchestration/stats`，形成可观测指标。
- 在执行链路中沉淀质量动作信息（`accept/revise/refuse`）到 `orchestration` 元数据。
- 将 P2 评测脚本升级为可自动输出 Markdown 报告。

## 本阶段改动

### 1) AgentV2 增加执行追踪（trace）

文件：`animal/src/agents/agent_v2.py`

- 新增 `_last_trace` 存储最近一次 `run_v2` 的执行追踪。
- 在 `run_v2` 的关键出口（闲聊直答、fallback_response、正常完成、异常）统一写入 trace。
- 新增：
  - `_build_execution_trace(context)`：汇总 `intent`、`reflection_rounds`、`tool_count`、`quality_action`、`quality_actions`。
  - `_collect_quality_actions(payload)`：递归提取工具结果中的 `quality_action`。
  - `get_last_trace()`：供上层编排读取追踪数据。

### 2) PetHealthAgent 将质量动作写入 orchestration

文件：`animal/src/agents/pet_health_agent.py`

- V2 主路径中读取 `self.agent_v2.get_last_trace()`，并写入：
  - `quality_action`
  - `quality_actions`
  - `intent`
  - `tool_count`
  - `reflection_rounds`
- `emergency_gate` / `v1 fallback` 路径补齐默认字段：
  - `quality_action: None`
  - `quality_actions: []`

### 3) 编排统计接口新增质量动作维度

文件：`animal/src/api/v1/endpoints/chat.py`

- chat 结构化日志增加 `quality_action` 字段。
- `/chat/orchestration/stats` 新增计数：
  - `counters.by_quality_action = {accept, revise, refuse, unknown}`
- `rates` 新增比率：
  - `quality_accept_rate`
  - `quality_revise_rate`
  - `quality_refuse_rate`

### 4) 自动化测试补充质量维度断言

文件：`animal/tests/test_chat_api.py`

- `test_chat_orchestration_stats_endpoint` 造数时加入 `quality_action`。
- 增加断言：
  - `counters.by_quality_action.accept/revise/refuse`
  - `rates.quality_accept_rate/quality_revise_rate/quality_refuse_rate`

### 5) 评测脚本输出 Markdown 报告

文件：`animal/scripts/run_p2_smoke_eval.py`

- 新增 `render_markdown_report()`：生成质量门禁与安全后置校验的评测结论。
- 新增 `write_report()`：自动写入 `animal/docs/P2第三阶段评测报告_YYYYMMDD.md`。
- 运行脚本后会在终端打印报告路径。

## 验证建议

1. 运行单测：

```bash
pytest animal/tests/test_chat_api.py -k orchestration_stats -q
```

2. 运行评测脚本：

```bash
python animal/scripts/run_p2_smoke_eval.py
```

3. 检查输出文档：

- `animal/docs/P2第三阶段评测报告_YYYYMMDD.md`

## 预期收益

- 编排统计从“引擎层可见”提升为“质量动作可见”，可量化观察拒答与修正趋势。
- 评测结果可沉淀为文档资产，便于发布前审查与阶段复盘。
