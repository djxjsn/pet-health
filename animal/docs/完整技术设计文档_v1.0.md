## 1. 项目整体架构

### 1.1 核心业务目标

为宠物主人提供基于AI的智能健康咨询服务，通过RAG知识增强、Agent工具调用、MCP协议集成，解决以下痛点：

| 痛点 | 解决方案 | 优先级 |
|------|---------|--------|
| 宠物健康咨询难 | AI智能症状分析+RAG知识检索 | P0 |
| 多宠物管理混乱 | 宠物档案+上下文感知对话 | P0 |
| 宠物用品选购决策困难 | 智能购物助手+成分分析 | P1 |

### 1.2 3个最主要用户场景

| 场景编号 | 用户场景 | 触发条件 | 核心流程 |
|---------|---------|---------|---------|
| UC-01 | 宠物健康咨询 | 宠物出现异常症状 | 描述症状→Agent识别意图→RAG检索知识→调用工具分析→生成建议 |
| UC-02 | 宠物档案管理 | 养新宠物/查看历史记录 | 创建档案→设置当前宠物→上下文注入→个性化对话 |
| UC-03 | 智能商品推荐 | 对话中推荐/主动查询 | 健康状态分析→商品搜索→成分适配→生成推荐报告 |

### 2.3 前端功能模块输入输出

#### 模块1: 认证模块 - AuthModal

| 属性 | 详情 |
|------|------|
| 所属页面 | 登录页 `/login`、注册页 `/register` |
| 核心功能 | 用户注册、登录、密码重置 |

**输入参数**:
- 用户输入:
  - `phone`: string, 必填, 正则 `^1[3-9]\d{9}$`, 默认空, 错误提示 "请输入正确的手机号"
  - `email`: string, 选填, 正则邮箱格式, 默认空, 错误提示 "邮箱格式不正确"
  - `password`: string, 必填, 6-20字符, 默认空, 错误提示 "密码长度6-20位"
- 接收参数: `redirectUrl` (from query string)
- 存储读取: `auth_token` from localStorage (自动登录)

**输出参数**:
- 展示: 用户信息、登录状态
- 传递: `access_token`, `refresh_token`, `user_id`
- 存储写入: `auth_token` (localStorage), `current_user` (sessionStorage)

#### 模块2: 对话模块 - ChatContainer

| 属性 | 详情 |
|------|------|
| 所属页面 | 对话首页 `/chat` |
| 核心功能 | 实时AI对话、消息展示、流式响应 |

**输入参数**:
- 用户输入: `messageContent` (string, 必填, max 2000字符), `imageUrls` (string[], 可选, max 3)
- 接收参数: `conversation_id` (from route), `pet_id` (from petStore)
- 存储读取: `current_pet_id` (usePetStore), `auth_token` (useAuthStore)

**输出参数**:
- 展示: `messages[]` (role, content, timestamp, sources), `typingIndicator` (boolean)
- 传递: `new_message` → API, `conversation_id` → URL更新
- 存储写入: `chat_history:{conversation_id}` (sessionStorage)

#### 模块3: 宠物管理模块 - PetCard / PetForm

| 属性 | 详情 |
|------|------|
| 所属页面 | 宠物列表页 `/pets`、详情页 `/pets/[id]` |
| 核心功能 | 宠物档案CRUD、列表展示、宠物切换 |

**输入参数**:
- 用户输入: `name` (必填, max 50字符), `species` (必填, enum: cat/dog/other), `breed` (必填), `birth_date` (选填), `gender` (选填), `weight` (选填, 0-100)
- 接收参数: `pet_id` (from route for detail/edit)
- 存储读取: `auth_token` (useAuthStore)

**输出参数**:
- 展示: `pet_list[]` (pet_id, name, species, breed, age, weight, avatar_url, is_active)
- 传递: `selected_pet_id` → usePetStore
- 存储写入: `current_pet_id` (usePetStore), `pet_list_cache` (localStorage)

#### 模块4: 消息气泡模块 - MessageBubble

| 属性 | 详情 |
|------|------|
| 所属页面 | 对话首页 `/chat`、历史对话页 `/history` |
| 核心功能 | 消息展示、引用来源、操作按钮 |

**输入参数**:
- 接收参数: `role` (user/assistant), `content` (string), `timestamp` (Date), `sources` (KnowledgeSource[], 可选), `attachments` (Attachment[], 可选), `isStreaming` (boolean)

**输出参数**:
- 展示: 消息内容(支持Markdown)、引用来源卡片、操作按钮(复制/反馈)
- 传递: `feedback_score` → API, `copied_text` → 剪贴板

| 属性 | 值 |
|------|-----|

| MySQL字符集 | utf8mb4 / utf8mb4_unicode_ci |

**命名规范**:
- 表名: 小写蛇形命名, 复数形式 (如: `users`, `health_records`)
- 字段名: 小写蛇形命名 (如: `user_id`, `birth_date`)

- 外键: `关联表名_id` (如: `user_id`, `pet_id`)
- 索引名: `idx_字段名` (普通索引), `uk_字段名` (唯一索引), `fk_本表_关联表` (外键)

**分库分表策略**: 当前阶段不分库不分表。单表数据量预计<100万条。如增长到千万级, 采用按 `user_id` 分片策略。

### 6.5 Agent工作流程

**1. 意图识别逻辑**:
```
用户输入 → LLM分类(SYSTEM: "将以下输入分类为: health_consultation, behavior_analysis, shopping_recommendation, casual_chat")
         → 返回: {intent: "health_consultation", confidence: 0.95, slots: {symptom: ["软便"], pet_mentioned: false}}
         → 如confidence < 0.7, 使用规则引擎二次确认(关键词匹配)
```

**2. 工具选择与调用逻辑**:
```
意图 = health_consultation
  → 加载宠物档案(如未指定则使用active pet)
  → 调用 KnowledgeRetriever(query=symptoms, category="disease")
  → 调用 SymptomAnalyzer(symptoms, pet_profile, rag_results)
  → 调用 UrgencyEvaluator(symptoms, description)
  → 如有图片 → 调用 ImageRecognitionTool(image_urls)
  → 整合所有工具结果
```

**3. 结果整合与生成逻辑**:
```
组装Prompt:
  [System Prompt]
  [Pet Context]: 当前宠物档案信息
  [Memory]: 最近3轮对话历史
  [RAG Results]: Top-5相关知识文档
  [Tool Results]: 工具分析结果
  [User Query]: 用户当前输入

→ GPT-4o流式生成 → 逐token返回前端
```

**4. 异常处理逻辑**:
```
工具调用失败:
  1. 重试1次(间隔1秒)
  2. 仍失败 → 降级: 仅使用RAG结果生成回答
  3. 返回: "暂时无法获取实时信息, 基于现有知识为您分析..."

LLM API失败:
  1. 切换备用模型(如GPT-4o → DeepSeek)
  2. 仍失败 → 返回预设回复 + 记录错误

RAG检索为空:
  1. 放宽相似度阈值(0.7 → 0.5)
  2. 仍无结果 → 仅依赖LLM内建知识 + 免责声明
```
