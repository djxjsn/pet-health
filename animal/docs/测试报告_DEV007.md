## 5. 缺陷记录

### 5.1 已修复问题

| 编号 | 问题 | 严重程度 | 状态 |
|------|------|----------|------|
| BUG-001 | Pydantic模型缺少必填字段导致序列化失败 | 高 | ✅ 已修复 |
| BUG-002 | patch.object 使用错误导致 mock 失效 | 中 | ✅ 已修复 |
| BUG-003 | AllergenWarning 字段不匹配 | 中 | ✅ 已修复 |

### 5.2 修复方案

**BUG-001**: 为所有 ProductResponse mock 数据补充完整字段：
```python
{
    "_id": "obj_xxx",
    "product_id": "prod_xxx",
    # ... 其他字段 ...
    "subcategory": None,
    "ingredients": [],
    "nutrition_info": {},
    "suitable_for": [],
    "tags": [],
    "review_count": 0,
    "currency": "CNY"
}
```

**BUG-002**: 将 `patch.object(service, 'ProductRepository')` 改为 `patch("src.services.shopping_service.ProductRepository")`

**BUG-003**: 补充 allergen_warnings 的完整字段结构
